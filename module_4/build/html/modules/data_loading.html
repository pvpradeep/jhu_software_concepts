Data Loading and Database Management

src.load_data.create_table()

Create the database schema for storing grad records.

Creates three tables in the database: - grad_records_common: Base table with common schema (parent table) - gradrecords: Main table inheriting from common (stores all records) - gradrecords_latest: Table for most recent entries (also inherits from common)

The function uses table inheritance to maintain schema consistency across tables. All tables share the same structure defined in create_common_sql.

Global variables:

  • pool: Database connection pool used for executing SQL

src.load_data.fetch_new_data(pool)

Orchestrate the complete process of fetching and processing new grad records.

This function manages the entire pipeline of: 1. Scraping new data from GradCafe 2. Processing the data through the LLM for entity extraction 3. Converting data formats (JSON -> JSONL -> cleaned JSON) 4. Loading processed data into the database 5. Cleaning up temporary files

Parameters:

pool (psycopg_pool.ConnectionPool) – Database connection pool for database operations

Returns:

None

Global variables:

  • JSON_DATA_FILEPATH: Directory for temporary and permanent data files

src.load_data.get_grad_records_latest()

Retrieve the most recent graduate records for comparison.

This function fetches all records from the gradrecords_latest table and converts them into a list of dictionaries. This format is used for comparing against newly scraped records to avoid duplicates.

Returns:

List of most recent graduate records

Return type:

list[dict]

Global variables:

  • pool: Database connection pool

Note:

The returned dictionaries preserve the database column names as keys, making them directly comparable with scraped JSON data.

src.load_data.init_db()

Initialize and set up the complete database environment.

This function performs the complete database initialization sequence: 1. Resets any existing database structure 2. Creates new tables 3. Loads initial data 4. Prints summary statistics and query results

The function uses a connection pool that remains open until program termination (handled by atexit).

Returns:

Initialized database connection pool

Return type:

psycopg_pool.ConnectionPool

Global variables:

  • pool: Database connection pool

src.load_data.insert_grad_records(json_data)

Insert grad records into both the main and latest tables.

This function coordinates the insertion of records into both the main gradrecords table and the gradrecords_latest tracking table. It ensures the most recent entries are available for comparison while maintaining the complete dataset.

Parameters:

json_data (list[dict]) – List of grad record entries to insert

Global variables:

  • pool: Database connection pool

  • MAX_MARKERS: Used indirectly via insert_grad_records_latest

src.load_data.insert_grad_records_latest(json_data)

Update the gradrecords_latest table with the most recent entries.

This function maintains a small set of the most recent records for quick comparison during scraping. It first clears the existing entries, then inserts up to MAX_MARKERS entries from the new data.

Parameters:

json_data (list[dict]) – List of grad record entries to process

Global variables:

  • pool: Database connection pool

  • MAX_MARKERS: Maximum number of recent records to keep

src.load_data.load_new_data_to_db(latest_file)

Process newly scraped data and add it to the database.

This function handles the integration of new data files by: 1. Finding the next available index in the sequence of data files 2. Renaming the temporary file to match the sequence 3. Processing the renamed file’s contents into the database

Parameters:

latest_file (str) – Path to the temporary file containing new data

Returns:

Number of records processed from the file

Return type:

int

Global variables:

  • JSON_DATA_FILEPATH: Directory for data files

  • JSON_DATA_FILENAME: Base name for sequenced files

src.load_data.process_json_files()

Process all JSON data files in sequence and load them into the database.

Searches for files matching the pattern JSON_DATA_FILENAME_{i}.json where i starts from 0 and increments until no more files are found. Each file’s contents are processed and inserted into the database.

Returns:

Total number of records processed across all files

Return type:

int

Global variables:

  • JSON_DATA_FILEPATH: Base directory for JSON files

  • JSON_DATA_FILENAME: Base filename pattern for JSON files

src.load_data.process_one_json(filename)

Process a single JSON file and insert its records into the database.

Reads the specified file from the JSON_DATA_FILEPATH directory, processes its contents, and inserts the records into the database. Handles file not found errors gracefully.

Parameters:

filename (str) – Name of the JSON file to process

Returns:

Total number of records processed

Return type:

int

Global variables:

  • JSON_DATA_FILEPATH: Directory containing the JSON files
